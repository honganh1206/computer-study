A central LLM dynamically breaks down tasks, delegates/routes them to worker LLMs and synthesizes their results

![[Pasted image 20250624175033.png]]

When? Complex tasks where we are not sure of the subtasks needed e.g., number of files to be changed. The orchestrator determines the subtasks

Examples:

- Coding products that make complex changes to multiple files each time.
- Search tasks that involve gathering and analyzing information from multiple sources for possible relevant information.
